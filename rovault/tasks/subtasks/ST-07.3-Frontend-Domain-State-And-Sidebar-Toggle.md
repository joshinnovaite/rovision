# ST-07.3: Frontend Domain State + Sidebar Toggle

Parent: [[BOW-07-Multi-Domain-App-Architecture]]
Status: done
Progress: done — domain state in `videoStore` (`setDomain`/`loadDomains`), `resetForDomain` in `settingsStore`, sticky sidebar segmented toggle (localStorage); typechecks + builds.

## Preconditions
- Backend exposes `/api/domains` and domain params ([[ST-07.2-Backend-Domain-Plumbing]]).
- The existing frontend foundation ([[BOW-03-Frontend-Foundation]]) and sidebar ([[ST-04.1-Sidebar-And-Library]]) exist.

## Steps
1. `types/index.ts`: add `domain`/`mode` and registry-payload types.
2. `lib/api.ts`: add `getDomains`; thread `domain` into config/videos calls.
3. `lib/classColors.ts`, `lib/assets.ts`: source colours/lenses from the active domain's registry block (drop hardcoded copies).
4. `state/videoStore.ts`: add domain state + `setDomain` / `loadDomains`; filter the library by domain.
5. `state/settingsStore.ts`: `resetForDomain` to clear/seed per-domain UI settings on toggle.
6. `components/sidebar/Sidebar.tsx`: persistent segmented control at the top; selection sticky via localStorage.
