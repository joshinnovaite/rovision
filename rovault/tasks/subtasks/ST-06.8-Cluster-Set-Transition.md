# ST-06.8: Cluster-Set Transition

Parent: [[BOW-06-Horizontal-LOD-Playback-Bar]]
Status: done
Progress: done — cluster-set transition kept **intentionally minimal**: **hover-protrude only** (CSS transform); card positions **snap** on zoom (deterministic, no flicker). Full **FLIP position animation was deliberately deferred** to avoid pinch-smear — recorded here as future polish. Visual confirmation pending.

## Preconditions
- [[ST-06.3-Deterministic-Cluster-Algorithm]] done (clusters recompute per zoom level).
- [[ST-06.7-Max-Zoom-Deck-Of-Cards-Hover]] done (hover-protrude affordance).

## Steps
1. On a zoom step the cluster set changes; let card positions **snap** to the new deterministic layout
   (no membership churn → no flicker, so snapping is acceptable).
2. Animate **only hover-protrude** (CSS transform) for the deck-of-cards / cluster hover.
3. **Decision: defer full FLIP** (First-Last-Invert-Play) position animation between cluster sets.
   Reason: animating positions *during a continuous pinch* produces visible **smear**; the deterministic
   snap is clean and cheap for the demo. FLIP is logged as **future polish**, not a bug.
