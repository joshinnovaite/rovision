# ST-10.10: Sky/Water Aggregate Scene Badge (meta.json + Backend Read + SceneBadge)

Parent: [[BOW-10-E1-Phase-0-Proof-Of-Capability]]
Status: done
Progress: done (recorded retroactively) — scene state surfaced as a **per-clip aggregate badge** carried in
the bundle's `meta.json` (`scene: {sky_state, water_state, method, metrics}`), read by the backend in
`GET /api/videos/{hash}` (**no schema migration** — meta.json read, not a DB column), and rendered by a new
data-driven **SceneBadge** in `VideoHeader.tsx` that appears **only when scene data is present** (subsea /
insulators untouched). See [[DR-019-Scene-Attribute-Aggregate-Badge-Surface]].

## Preconditions
- Calibrated thresholds from the §18b sweep ([[ST-10.9-S18b-Calibration-Sweep]]).
- Multi-domain backend/frontend already domain-aware ([[BOW-07-Multi-Domain-App-Architecture]]).

## Steps
1. Producer: emit a per-clip `scene` aggregate block into each E1 bundle's `meta.json`.
2. Backend: surface `scene` in `GET /api/videos/{hash}` (read meta.json; no schema change).
3. Frontend: add a data-driven `SceneBadge` to `VideoHeader.tsx`, rendered only when `scene` is present.

## Files
- `app/backend/data/bundles/<e1>/meta.json` — `scene: {sky_state, water_state, method, metrics}`.
- `app/backend/main.py` — `GET /api/videos/{hash}` surfaces `scene`.
- `app/frontend/.../VideoHeader.tsx` — new `SceneBadge` (conditional on scene data).

## Acceptance
- E1 clips show a sky/water badge sourced from meta.json; subsea/insulators clips render unchanged
  (no badge, no schema migration).
