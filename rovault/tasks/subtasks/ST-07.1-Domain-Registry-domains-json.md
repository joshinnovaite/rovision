# ST-07.1: Domain Registry (domains.json)

Parent: [[BOW-07-Multi-Domain-App-Architecture]]
Status: done
Progress: done — `rovision/domains.json` authored; `subsea` block carries the unchanged 15-class taxonomy.

## Preconditions
- The four existing taxonomy copies are known (labeller, notebook §15, backend `config.py`, frontend `classColors.ts`/`assets.ts`).
- The subsea defect-subset, colours, asset lenses, and severity thresholds are known from `config.py`/the frontend utils.

## Steps
1. Create `rovision/domains.json` keyed by domain (`subsea`, `pylon`).
2. Per domain define: `mode` (`defect`/`inventory`), `all_classes`, `defect_classes` (subset; empty for pylon), per-class `colors`, `assets` (lenses), `severity` thresholds.
3. Populate the `subsea` block from the existing config **unchanged** (the [[DR-003-Defect-Taxonomy-Consolidation]] 15-class list).
4. Seed the `pylon` block (mode `inventory`, empty `defect_classes`) with a **provisional** class list from the user's initial proposal (5 classes: insulator, foundation, pylon_body, pylon_top, vibration_damper, with colours) — to be confirmed/refined against actual footage by [[ST-08.1-Pylon-Footage-And-Class-Proposal]].
