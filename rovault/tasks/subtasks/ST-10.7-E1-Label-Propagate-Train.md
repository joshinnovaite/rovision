# ST-10.7: E1 Label → Propagate → Train (Colab GPU)

Parent: [[BOW-10-E1-Phase-0-Proof-Of-Capability]]
Status: done
Progress: done — E1 golden set labelled (31 records, QC'd) → propagated → YOLO11n trained on Colab GPU
(autocast-guarded per [[DR-015-Leaked-Global-Autocast-Corrupted-Detector-Training]]). The trained E1
detector is the Phase-0 carry-forward asset.

## Preconditions
- E1 clips staged in `test_footage_e1/` ([[ST-10.5-E1-Footage-Capture]]).
- E1 registered ([[ST-10.1-E1-Domain-Registration]]); labeller `--domain e1` appends to `labels_e1.json`.
- §16 autocast guard verified ([[ST-10.3-DR015-Autocast-Guard-Verified-S16]]) so the zero-mAP collapse
  ([[DR-015-Leaked-Global-Autocast-Corrupted-Detector-Training]]) cannot recur.

## Steps
1. Label an E1 golden set (`race_boat` / `turn_buoy` / `spray_plume`) via the labeller `--domain e1`.
2. Amplify: propagate seed boxes through short clips (§13/§14) into the E1 amplified dataset.
3. Convert to YOLO format (§15) and train a separate E1 YOLO11n (§16, overfit-by-design, autocast-guarded).
4. Sanity-check on footage (overfit metrics are inflated — look at real frames).

## Files
- `rovision/labelling/labels_e1.json` — E1 golden set.
- `rovision/dataset/` — E1 amplified data (gitignored; durable copy on Drive).
- `rovision/notebooks/subsea_defect_demo.ipynb` — §13–§16 run with `DOMAIN='e1'`.

## Acceptance
- A trained E1 YOLO11n exists and detects the E1 classes on real frames — the Phase-0 carry-forward asset.
