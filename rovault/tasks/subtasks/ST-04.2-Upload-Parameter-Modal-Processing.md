# ST-04.2: Upload, Parameter Modal, Processing

Parent: [[BOW-04-Shell-And-Pre-Playback-Screens]]
Status: done
Progress: done — drop-zone + ParameterModal (detector/SAM2 frequency sliders, Asset select, omitted-class chips). Submit -> /api/upload (sha256) -> on hit apply settings + load + route to dashboard; on miss show "process offline" message.

## Preconditions
- [[ST-03.3-Stores-And-Derived-Structures]] done (`settingsStore` exists).
- `POST /api/upload` ([[ST-02.5-Upload-Seed-And-Cache-Headers]]) available for live mode.

## Steps
1. Build the `/upload` screen with a file picker / drop zone.
2. Build the parameter modal: **asset** selector (single concept, replaces "footage type") plus the
   display-only frequency sliders (`detectorEveryN`, `refineEveryN`) — written to `settingsStore`.
3. On submit, `POST /upload` streaming the file; show a **processing** state while hashing.
4. On **hit** (200): load the matched hash into `videoStore`, route to `/v/:hash/dashboard`.
5. On **miss** (202): show a graceful "not in cache" message (no live inference exists).
6. Persist the chosen asset into `settingsStore` (changeable later on playback).
7. Make clear the **cache key is the hash only** — modal params never affect it (display-only).

## Files
- `app/frontend/src/routes/Upload.tsx`, `app/frontend/src/components/ParameterModal.tsx`.

## Acceptance
- Uploading the seeded `raw.mp4` hits cache (200) and routes to the dashboard with the chosen asset set.
- An unknown file shows the graceful miss state; sliders/asset write to `settingsStore`; the hash is
  unaffected by modal params.
