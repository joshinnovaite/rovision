# ST-06.4: Recompute On Zoom, Imperative Pan/Play

Parent: [[BOW-06-Horizontal-LOD-Playback-Bar]]
Status: done
Progress: done — clusters recompute **only on zoom-scale change**; pan (`viewStartTime`) and playback are pure **CSS transforms applied imperatively** via a `playbackStore` subscription — **zero React re-render on play / scrub / pan** (same discipline as the [[ST-05.2-Frame-Sync-Mask-Hold-Downsample|OverlayLayer]] / [[DR-009-Mask-Hold-Rendering]]). Visual confirmation pending.

## Preconditions
- [[ST-06.2-Continuous-Zoom-Gestures]] done (`scale` updates on zoom).
- [[ST-06.3-Deterministic-Cluster-Algorithm]] done (the recompute target).

## Steps
1. **Memoise** clusters keyed on **`scale`** (+ track set / `barWidthPx`) — recompute **only** when
   scale changes, never on pan or playback.
2. Drive the **playhead / pan** as a **CSS transform** (e.g. `translateX`) applied **imperatively** to
   the track container via `playbackStore.subscribe` — do **not** route per-frame motion through React
   state.
3. Map `currentTime` and `viewStartTime` → transform with `scale`; update the ref directly each frame.
4. Verify with the React profiler: **no component re-render** during play / scrub / pan; re-render
   occurs only on a zoom step.

## Files
- `app/frontend/src/components/playback/PlaybackBarCard.tsx` (memoised clusters + imperative subscribe),
  `app/frontend/src/state/playbackStore.ts` (`scale` / `viewStartTime` / `playbackRate`).

## Acceptance
- Panning and playback move the bar with **no React re-render**; clusters recompute only when zoom
  scale changes; the bar stays smooth under continuous gestures.
