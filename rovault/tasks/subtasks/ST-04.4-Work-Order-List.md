# ST-04.4: Work-Order List

Parent: [[BOW-04-Shell-And-Pre-Playback-Screens]]
Status: done
Progress: done — full-width WorkOrderList of expandable rows (action + count + severity pill; expand shows contributing instances with Jump-to-playback links via ?select=trackId).

## Preconditions
- [[ST-04.3-Dashboard-Metric-Cards]] done (`lib/severity.ts` exists, dashboard renders).

## Steps
1. Create `app/frontend/src/lib/workorders.ts` mapping defect class → action text:
   corrosion → "Remove corrosion / surface treatment"; coating_breakdown → "Repair / recoat
   affected area"; surface_deposit → "Clean surface deposits / sediment removal"; marine_growth →
   "Biofouling removal (cleaning campaign)"; trash_rack_blockage → "Clear blockage / debris
   removal"; seal_joint_degradation → "Inspect & reseal / replace joint"; dropped_object →
   "Investigate & remove foreign object".
2. Derive work orders from flagged tracks (severity from instance-count + area, via `lib/severity.ts`).
3. Build a **full-width** work-order list of compact rows below the metric cards.
4. Rows **expand** to show detail; keep work orders **generic** (not asset-stamped).
5. Each row links to its originating track: jump → `/v/:hash/playback` + select that `track_id`.
6. Recompute live with omittedClasses/asset filters (shared with the dashboard).

## Files
- `app/frontend/src/lib/workorders.ts` — class → action text map + severity-driven ordering.
- `app/frontend/src/components/WorkOrderList.tsx`.

## Acceptance
- A full-width list of expandable work-order rows renders from the fixture; action text matches the map.
- Expanding shows detail; a jump link navigates to playback with the right track selected; filters
  apply live; orders are generic (no asset stamp).
