# ST-07.2: Backend Domain Plumbing

Parent: [[BOW-07-Multi-Domain-App-Architecture]]
Status: done
Progress: done — registry loaded; `videos.domain` column + startup migration; ingest reads `meta.domain`; rollup yields zero flags for empty `defect_classes`; `/api/domains` + domain params added. Migration verified against an existing catalog.

## Preconditions
- `rovision/domains.json` exists ([[ST-07.1-Domain-Registry-domains-json]]).
- The existing backend ([[BOW-02-Backend]]) and its `videos`/`tracks` schema and rollup are in place.

## Steps
1. `config.py`: load `rovision/domains.json`; expose per-domain taxonomy/defect-subset/colours/assets/severity (replacing hardcoded constants).
2. `schema.sql`: add `domain` column to `videos` (`NOT NULL DEFAULT 'subsea'`).
3. `db.py`: in-place **startup migration** — add the column if absent so existing catalogs upgrade; legacy rows default to `subsea`.
4. `ingest.py`: read `"domain"` from bundle `meta.json` (default `subsea`); store on the row.
5. `rollup.py`: drive flags off the domain's `defect_classes` — empty (pylon) ⇒ `compute_rollup`/`isFlag` yield zero flags naturally.
6. `main.py`: add `/api/domains`; accept `domain` param on `/api/config` and `/api/videos`.

## Files
- `app/backend/config.py`, `schema.sql`, `db.py`, `ingest.py`, `rollup.py`, `main.py`.

## Acceptance
- Fresh and pre-existing DBs both end with a `domain` column (verified). `/api/domains` lists the registry; `/api/config?domain=` and `/api/videos?domain=` filter/select correctly; a pylon-like empty `defect_classes` produces zero flags.
