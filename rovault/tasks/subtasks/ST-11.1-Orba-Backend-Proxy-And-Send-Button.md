# ST-11.1: Orba Backend Proxy And Send Button

Parent: [[BOW-11-Orba-CMMS-Service-Request-Integration]]
Status: done
Progress: done ‚Äî backend proxy module + route + config, frontend `sendToOrba` + SendToOrbaButton wired into the Dashboard; verified against a local stub.

## Preconditions
- [[ST-04.4-Work-Order-List]] done (work-order findings derivable per video).
- The Orba developer's `orba/orbaAdapter.ts` available as the **contract reference** (kept, not imported).

## Steps
1. Port the adapter's pure mapping + sequential POST to `app/backend/orba.py` (severity‚Üípriority
   high‚Üí2 / medium‚Üí4 / low‚Üí6 / none‚Üískip; one SR per finding, sent sequentially via `httpx`).
2. Add `POST /api/orba/service-requests` in `main.py` (Pydantic `OrbaFinding`/`OrbaSendRequest`;
   **503** when `ORBA_BASE_URL`/`ORBA_ASSETNUM` unset).
3. Load `ORBA_BASE_URL`/`ORBA_INGEST_SECRET`/`ORBA_ASSETNUM` in `config.py` from a gitignored
   `app/backend/.env` (python-dotenv); commit `.env.example`.
4. Frontend: `sendToOrba` in `lib/api.ts`, `OrbaResult` in `types/index.ts`.
5. Build `components/dashboard/SendToOrbaButton.tsx` and wire it into `DashboardScreen.tsx`
   (explicit send; per-video dedup guard disables after a successful send for that video's hash).
6. Add `httpx`, `python-dotenv` to `app/backend/requirements.txt`.

## Files
- `app/backend/orba.py` ‚Äî severity‚Üípriority map + sequential `httpx` POST.
- `app/backend/main.py` ‚Äî `POST /api/orba/service-requests` route.

- `app/frontend/src/lib/api.ts` (`sendToOrba`), `types/index.ts` (`OrbaResult`),
  `components/dashboard/SendToOrbaButton.tsx`, `screens/DashboardScreen.tsx`.

## Acceptance
- Clicking "Send to Orba" posts one SR per finding to the same-origin route; backend forwards
  sequentially; button disables after success for that video; route returns 503 if env unset.
- Verified end-to-end against a **local stub** standing in for Orba.
