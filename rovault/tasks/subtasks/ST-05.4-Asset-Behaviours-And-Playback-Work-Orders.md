# ST-05.4: Asset Behaviours And Playback Work-Orders

Parent: [[BOW-05-Playback]]
Status: done
Progress: done — AssetDropdown (filter/prioritise via dimming in overlay+cards, auto-jump to top-priority/most-prominent class) + WorkOrderChips (lighter playback work orders, click -> jump to first instance). Dashboard work-order rows also jump via ?select=trackId.

## Preconditions
- [[ST-05.3-Vertical-Timeline]] done (timeline cards + selection working).
- `lib/workorders.ts` from [[ST-04.4-Work-Order-List]] exists.

## Steps
1. Create `app/frontend/src/lib/assets.ts` with the asset → `relevantClasses` / `topPriority` map
   (gold_coast_desal, hinze, little_nerang, north_pine, wivenhoe, somerset, generic).
2. Add an **asset dropdown** on playback bound to `settingsStore.asset` (changeable live).
3. Behaviour — **filter / prioritise** cards by `relevantClasses` for the selected asset.
4. Behaviour — **auto-jump** to the asset's `topPriority` class (seek to its first qualifying track).
5. Behaviour — **de-emphasise (dim)** off-asset classes in overlay + timeline.
6. Surface **work-order chips** lighter on the playback screen (from `lib/workorders.ts`).
7. Keep flags = all 15 classes with artefacts dimmed by default; asset selection dims further.

## Files
- `app/frontend/src/lib/assets.ts` — `relevantClasses` + `topPriority` per asset.
- `app/frontend/src/components/AssetDropdown.tsx`, playback work-order chip component.

## Acceptance
- Changing the asset dropdown live re-filters/prioritises cards, auto-jumps to the top-priority
  class, and dims off-asset classes in overlay + timeline.
- Work-order chips appear lighter on playback; concrete cracking/spalling absent (no such class).
