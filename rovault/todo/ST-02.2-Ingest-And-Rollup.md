Parent: [[BOW-02-Backend]]

## Preconditions
- [[ST-02.1-Skeleton-Config-Schema-Models]] done: `schema.sql`, `db.py`, `config.py` exist.
- A real bundle from [[ST-01.2-Raw-Clip-Encode-And-Fixture-Capture]] is available on disk.

## Steps
1. Create `app/backend/ingest.py` with a CLI entry: `python -m app.backend.ingest <bundle_dir>`.
2. Read `meta.json`, `detection_cards.json`, `track_cards.json` from the bundle dir.
3. Compute `hash = sha256(raw.mp4 bytes)` — hash the **exact** `raw.mp4` (same bytes the user uploads).
4. `INSERT OR REPLACE` the `videos` row (meta fields + `bundle_dir`, `status`, `processed_at`).
5. `INSERT OR REPLACE` `tracks` rows from `track_cards.json` (one per `track_id`).
6. Compute the per-video **rollup** using `config.py`: `flag_count` (tracks past
   `FLAG_MIN_N_FRAMES` AND `FLAG_MIN_PEAK_AREA`), `max_severity` (from instance counts +
   `SEV_COVERAGE_BUMP`); write back onto the `videos` row.
7. Copy the bundle directory to `data/static/<hash>/`.
8. Make it idempotent on `hash`: re-ingesting the same bundle replaces rows, does not duplicate.

## Files
- `app/backend/ingest.py` — `ingest_bundle(bundle_dir: str) -> str` (returns hash);
  `compute_rollup(tracks, config) -> (flag_count, max_severity)`.

## Acceptance
- `python -m app.backend.ingest <bundle>` populates `videos` (1 row, correct rollup) + `tracks`
  (N rows) and copies the bundle to `data/static/<hash>/`.
- Re-running on the same bundle yields the same `hash` and no duplicate rows (idempotent).
