# ST-02.1: Skeleton, Config, Schema, Models

Parent: [[BOW-02-Backend]]
Status: done
Progress: done — app/backend/{config.py,schema.sql,db.py} + requirements.txt; venv at app/.venv.

## Preconditions
- The bundle format from [[BOW-01-Notebook-Export-Extension]] is known (the ingest contract).
- Python env with FastAPI + uvicorn available (backend runs locally on the Mac; CPU only).

## Steps
1. Create `app/backend/` package (`__init__.py`, `main.py` mounting the FastAPI app on `/api`).
2. Create `app/backend/config.py`: `DEFECT_CLASSES` (6 defects + `dropped_object`);
   `FLAG_MIN_N_FRAMES ≈ 8`, `FLAG_MIN_PEAK_AREA ≈ 1500` (px); severity
   `SEV_MEDIUM_COUNT = 2`, `SEV_HIGH_COUNT = 4`, `SEV_COVERAGE_BUMP = 0.05`.
3. Create `app/backend/schema.sql` `videos` table: `hash PK, filename, asset, fps, enc_fps, width,
   height, n_frames, start_frame, refine_every, duration_sec, flag_count, max_severity, status,
   bundle_dir, processed_at`.
4. Add `schema.sql` `tracks` table: `hash, track_id, class, first_frame, last_frame, n_frames,
   peak_conf, peak_area_px`, `PRIMARY KEY(hash, track_id)`. (No detections table — served as blob.)
5. Add a DB module (`db.py`) that opens `data/rovision.db` and applies `schema.sql` on startup.
6. Define Pydantic response models (`models.py`) for video rows, track rows, and config payload.
7. Establish the static layout: `app/backend/data/static/<hash>/` for bundle dirs; gitignore `data/`.
8. Wire `uvicorn app.backend.main:app --port 8000`; add a CORS/dev-proxy note for vite :5173.
