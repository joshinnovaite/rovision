# ST-04.1: Sidebar And Library

Parent: [[BOW-04-Shell-And-Pre-Playback-Screens]]
Status: done
Progress: done — two-state sidebar (New upload + section nav Dashboard/Playback when a video is loaded + expandable Video library); AppShell wires it. Library click loads + routes.

## Preconditions
- [[ST-03.3-Stores-And-Derived-Structures]] done (stores + router shell exist).
- `GET /api/videos` ([[ST-02.3-Read-Endpoints]]) or fixture provides the library list.

## Steps
1. Build the sidebar component, state **derived from `videoStore.hash != null`**.
2. **Default state** (no video loaded): New-upload entry + Video-library (collapsed).
3. **Loaded state** (video present): New-upload + Section-nav (dashboard/playback) + Video-library.
4. Build the Video-library list from `GET /videos` rows (rollup tiles: flag_count, max_severity).
5. Clicking a library item loads that hash into `videoStore` and routes to `/v/:hash/dashboard`.
6. Style with tokens (severity colour only on severity tiles).

## Files
- `app/frontend/src/components/Sidebar.tsx`, `app/frontend/src/components/VideoLibrary.tsx`.

## Acceptance
- With no video loaded the sidebar shows the default two-entry state; once a video loads it shows
  the three-entry state with section nav.
- Library tiles list ingested videos with rollup; clicking one navigates to its dashboard.
