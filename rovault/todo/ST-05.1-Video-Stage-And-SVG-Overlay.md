# ST-05.1: Video Stage And SVG Overlay

Parent: [[BOW-05-Playback]]
Status: done
Progress: done — VideoStage (raw <video> + native controls, source-aspect stage) + OverlayLayer SVG (viewBox = source px; boxes + normalized polygons; class colours; selected = teal non-scaling stroke; artefact/off-asset dimming). Code-complete; visual confirmation pending.

## Preconditions
- [[ST-03.3-Stores-And-Derived-Structures]] done (geometry utils, stores, `FrameIndex`).
- Clip available via `GET /clip?variant=raw` ([[ST-02.4-Range-Seekable-Clip-Serving]]) or fixture.

## Steps
1. Build the playback route `/v/:hash/playback` with a `<video>` element as the stage (the
   `<video>` owns `currentTime`).
2. Point the source at `/api/videos/{hash}/clip?variant=raw` (range-seekable).
3. Render **boxes** in a source-pixel-viewBox `<svg>` (viewBox = `0 0 width height` from meta).
4. Render **polygons** in a separate `0 0 1 1` `<svg>` (normalized) with
   `preserveAspectRatio` so resize needs no recompute.
5. Drive both overlays' styling via `data-selected` / `data-dim` attributes (no per-shape React state).
6. Wire selection visuals to a single `selectedTrackId` from `playbackStore`.

## Files
- `app/frontend/src/routes/Playback.tsx`, `app/frontend/src/components/VideoStage.tsx`,
  `app/frontend/src/components/OverlayLayer.tsx`.

## Acceptance
- The clip loads and plays; boxes + polygon masks render correctly aligned over the video.
- Resizing the window keeps overlays aligned without recompute (viewBox scaling).
- Selecting a track applies `data-selected` highlight to its box + polygon.
