# ST-06.7: Max-Zoom Deck-Of-Cards Hover

Parent: [[BOW-06-Horizontal-LOD-Playback-Bar]]
Status: done
Progress: done — at **maximum zoom** clustering terminates (every flag is a singleton); where flags still coincide too tightly to separate, a **"deck-of-cards" hover** protrudes the stacked cards (CSS transform) to reveal each. Pure hover affordance — no recompute, no re-render. Visual confirmation pending.

## Preconditions
- [[ST-06.3-Deterministic-Cluster-Algorithm]] done (singleton terminal state at max scale).
- [[ST-06.6-Density-Card-Expansion-Snap-To-Zoom]] done (snap reaches the terminal zoom).

## Steps
1. At **max `scale`**, clustering produces **all singletons** (the de-overlap relaxation has spread
   everything); there are no DensityCards left.
2. Where near-coincident flags still stack, render them as a **deck** and on **hover protrude** the
   stack (fan / offset via **CSS transform** only) so each card is reachable.
3. Keep it a **pure hover affordance** — no cluster recompute, no React re-render (transform only),
   consistent with [[ST-06.4-Recompute-On-Zoom-Imperative-Pan-Play]].
4. Hovering a card in the deck previews it; clicking selects its track.

## Files
- `app/frontend/src/components/playback/FlagCard.tsx` (deck/hover-protrude),
  `app/frontend/src/components/playback/PlaybackBarCard.tsx` (terminal max-zoom state).

## Acceptance
- At maximum zoom no density cards remain; a stack of coincident flags fans out on hover (CSS only)
  so each is individually selectable; no re-render on hover.
