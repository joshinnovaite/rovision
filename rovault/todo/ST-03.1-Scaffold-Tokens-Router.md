# ST-03.1: Scaffold, Tokens, Router

Parent: [[BOW-03-Frontend-Foundation]]
Status: done
Progress: done — Vite+React+TS scaffold at app/frontend (vite proxy /api->:8000), tokens.css (teal/severity/Inter), hash router with Upload/Dashboard/Playback + AppShell. tsc --noEmit + vite build pass.

## Preconditions
- Node + npm available. No backend required (this is pure scaffold).

## Steps
1. Scaffold `app/frontend` with Vite + React + TypeScript.
2. Add Lucide icons, Inter font; configure a **light theme** with teal/cyan accent.
3. Define design tokens (CSS vars / theme module): **severity colours reserved for severity only**.
4. Configure the Vite dev server on **:5173** with a **dev proxy** to the API on :8000 (`/api`).
5. Set up React Router as a **hash router** with routes `/upload`, `/v/:hash/dashboard`,
   `/v/:hash/playback`.
6. Add a root layout shell placeholder (sidebar slot + main content outlet).
7. Confirm `npm run dev` serves and HMR ("livereload") works.

## Files
- `app/frontend/` — `vite.config.ts` (dev proxy), `src/main.tsx`, `src/App.tsx` (router),
  `src/theme/tokens.css` (design tokens), `index.html`.

## Acceptance
- `npm run dev` serves on :5173 with HMR; the three routes resolve (hash router).
- Design tokens load; teal/cyan accent + Inter render; severity palette is isolated.
