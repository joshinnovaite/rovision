# ST-02.3: Read Endpoints

Parent: [[BOW-02-Backend]]
Status: done
Progress: done — /api/config,/videos,/videos/{hash},/tracks,/detections all verified.

## Preconditions
- [[ST-02.2-Ingest-And-Rollup]] done: at least one video + its tracks are in `rovision.db` and the
  bundle is copied to `data/static/<hash>/`.

## Steps
1. `GET /api/videos` — library list of `videos` rows + rollup; support filter/sort query params.
2. `GET /api/videos/{hash}` — single video meta + rollup (404 if absent).
3. `GET /api/videos/{hash}/tracks` — the `tracks` rows for that hash.
4. `GET /api/videos/{hash}/detections` — return `detection_cards.json` from the bundle as a single
   blob (gzip; immutable cache headers added in ST-02.5).
5. `GET /api/config` — return `DEFECT_CLASSES` + the flag/severity thresholds from `config.py`
   (so the frontend shares one source of truth).
6. Wire all routes under the `/api` router from ST-02.1; return the Pydantic models.

## Files
- `app/backend/routes.py` (or `main.py` routers) — the five read endpoints.

## Acceptance
- `GET /api/videos` lists the ingested video with its rollup.
- `GET /api/videos/{hash}/tracks` returns the track rows; `GET /api/config` returns the thresholds.
- `GET /api/videos/{hash}/detections` returns the per-frame blob (the player can scrub from it).
