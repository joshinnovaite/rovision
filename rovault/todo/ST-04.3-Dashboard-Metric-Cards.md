# ST-04.3: Dashboard Metric Cards

Parent: [[BOW-04-Shell-And-Pre-Playback-Screens]]
Status: done
Progress: done — 4 equal MetricCards (Total flags, Classes detected w/ defect/artefact split, Most prominent, Severity summary) computed live via lib/severity from tracks + /api/config, honouring omitted-classes.

## Preconditions
- [[ST-03.3-Stores-And-Derived-Structures]] done (`videoStore.tracks`, `settingsStore` loaded).
- `GET /api/config` thresholds available (or fixture config).

## Steps
1. Create `app/frontend/src/lib/severity.ts`: compute flags + severity **client-side** from
   `tracks` + `/api/config` thresholds, so omitted-classes/asset filters apply live.
2. Build the dashboard route at `/v/:hash/dashboard` with **4 equal metric cards**:
   Total flags · Classes detected · Most prominent defect · Severity summary.
3. Derive each card from `lib/severity.ts` over the current `videoStore.tracks` filtered by
   `settingsStore` (omittedClasses, asset).
4. Use severity colours **only** on the severity summary card.
5. Recompute live when omittedClasses/asset change (no refetch — backend rollup is for tiles only).

## Files
- `app/frontend/src/lib/severity.ts` — flag/severity computation shared across screens.
- `app/frontend/src/routes/Dashboard.tsx`, `app/frontend/src/components/MetricCard.tsx`.

## Acceptance
- Dashboard shows 4 equal cards with correct values from the fixture.
- Toggling an omitted class or changing asset updates the cards live (no network refetch).
