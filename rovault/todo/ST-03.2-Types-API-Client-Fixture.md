# ST-03.2: Types, API Client, Fixture

Parent: [[BOW-03-Frontend-Foundation]]
Status: done
Progress: done — src/types, src/lib/api.ts, plus asset map (lib/assets.ts), severity (lib/severity.ts) and work-order (lib/workorders.ts) configs. Real backend used directly (dam_trash_rack), so no synthetic fixture needed.

## Preconditions
- [[ST-03.1-Scaffold-Tokens-Router]] done (Vite app + dev proxy + router exist).
- A fixture bundle captured by [[ST-01.2-Raw-Clip-Encode-And-Fixture-Capture]] is available.

## Steps
1. Define TypeScript types mirroring the bundle/API: `Meta`, `DetectionCard` (with optional
   `polygons`), `TrackCard`, `VideoRow`, `Config`.
2. Build `src/lib/api.ts` typed client wrapping `/api/videos`, `/{hash}`, `/tracks`,
   `/detections`, `/config`, `/upload`, and the `/clip?variant=` URL builder.
3. Commit the captured **fixture bundle** under `app/frontend/` (e.g. `public/fixture/<hash>/`) so
   BOW-03/04/05 can develop **without** a live backend.
4. Add a fixture-mode switch: api client reads the committed fixture when the backend is absent.
5. Verify types compile against the fixture JSON (no `any` leakage on card/polygon shapes).

## Files
- `app/frontend/src/types.ts` — `Meta`, `DetectionCard`, `TrackCard`, `VideoRow`, `Config`.
- `app/frontend/src/lib/api.ts` — typed client + clip URL builder.
- `app/frontend/public/fixture/<hash>/` — committed fixture bundle (small clip OK).

## Acceptance
- `tsc` passes; types match the fixture JSON shape (including optional `polygons`).
- The api client returns fixture data in fixture mode and real data when the backend is up.
